# 动静分离与分布式缓存说明

## 一、动静分离

### 1.1 实现说明

- **静态资源**：HTML、CSS、JS 存放在 `deploy/static/`
- **Nginx 配置**：`/static/` 由 Nginx 直接返回，设置 1 小时缓存
- **动态请求**：`/api/` 转发到后端服务

### 1.2 访问方式

| 地址 | 说明 |
|------|------|
| http://localhost/ | 首页（index.html） |
| http://localhost/static/index.html | 静态 HTML |
| http://localhost/static/css/style.css | 静态 CSS |
| http://localhost/api/users/ping | 动态 API |

### 1.3 JMeter 压测

```powershell
# 压测静态资源（预期响应时间更短）
jmeter -n -t "deploy/jmeter/static-test.jmx" -l static-results.jtl

# 压测动态 API
jmeter -n -t "deploy/jmeter/dynamic-test.jmx" -l dynamic-results.jtl
```

**观察点**：静态文件通常由 Nginx 直接返回，响应时间更短；动态 API 需经后端处理，响应时间略长。

## 二、分布式缓存（Redis）

### 2.1 商品详情页缓存

- **接口**：`GET /api/products/{id}`
- **实现**：ProductService 查询前先查 Redis，命中则直接返回，未命中则查 MySQL 并写入缓存

### 2.2 缓存问题处理

| 问题 | 处理方式 |
|------|----------|
| **缓存穿透** | 查询不存在的 id 时，将空结果写入 `product:null:{id}`，TTL 60 秒，避免反复查库 |
| **缓存击穿** | 热点 key 失效时，使用 Redis `SETNX` 分布式锁，只有一个线程查库并回填缓存 |
| **缓存雪崩** | 设置 TTL 时加随机偏移（3600 + 0~300 秒），避免大量 key 同时过期 |

### 2.3 验证缓存

```powershell
# 第一次请求：查库并写入缓存
curl.exe http://localhost/api/products/1

# 第二次请求：命中缓存（响应更快）
curl.exe http://localhost/api/products/1
```
