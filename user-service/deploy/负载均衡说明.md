# 负载均衡实现说明

## 一、架构说明

```
                    Nginx (80 端口)
                           |
                    负载均衡 (轮询)
                    /              \
            user-service-1      user-service-2
              (8081)              (8081)
```

- **Nginx**：监听 80 端口，作为统一入口，将请求转发到后端多个实例
- **user-service-1 / user-service-2**：两个独立实例，容器内均监听 8081，宿主机分别映射 8081、8082

## 二、启动方式

### 1. 构建并启动（Docker Compose）

```bash
cd user-service
docker compose up -d --build
```

启动后：

- Nginx: http://localhost
- user-service-1: http://localhost:8081（直连，用于对比）
- user-service-2: http://localhost:8082（直连，用于对比）

### 2. 验证负载均衡

```bash
curl -i http://localhost/api/users/ping
```

响应头中应看到 `X-Instance-Id: 1` 或 `X-Instance-Id: 2`，多次请求应交替出现。

## 三、Nginx 负载均衡算法

在 `deploy/nginx/nginx.conf` 中可切换算法：

| 算法 | 配置 | 说明 |
|------|------|------|
| **轮询（默认）** | 无额外配置 | 依次分配到 8081、8082 |
| **最少连接** | 添加 `least_conn;` | 优先分配给当前连接数最少的实例 |
| **IP 哈希** | 添加 `ip_hash;` | 同一客户端 IP 始终访问同一实例 |

## 四、JMeter 压测

### 1. 安装 JMeter

- 官网: https://jmeter.apache.org/download_jmeter.cgi
- 解压后运行 `bin/jmeter`（Windows 为 `bin/jmeter.bat`）

### 2. 使用已有测试计划

1. 启动：`jmeter`
2. 打开：`File` -> `Open` -> 选择 `deploy/jmeter/load-balance-test.jmx`
3. 运行：点击绿色三角形
4. 查看结果：`Summary Report` 中查看响应时间、吞吐量等

### 3. 手动创建测试计划

1. **Thread Group**：线程数 50，Ramp-up 5 秒，循环次数 100
2. **HTTP Request**：
   - Server: localhost
   - Port: 80
   - Path: /api/users/ping
   - Method: GET
3. **Listeners**：添加 Summary Report、View Results Tree

### 4. 验证请求分配

压测过程中，查看后端容器日志：

```bash
docker logs user-service-1 2>&1 | grep "Instance-1" | wc -l
docker logs user-service-2 2>&1 | grep "Instance-2" | wc -l
```

轮询模式下，两个实例的请求数应大致相等（误差在 10% 以内）。

## 五、日志格式

每个请求会输出类似：

```
[Instance-1] GET /api/users/ping
[Instance-2] GET /api/users/ping
```

便于在日志中统计各实例处理的请求数量。
