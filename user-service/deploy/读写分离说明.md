# MySQL 读写分离说明

## 一、架构

```
应用 (user-service / product-service)
    |
    +-- 写操作 --> MySQL Master :3307
    |
    +-- 读操作 --> MySQL Slave  :3308
                        |
                        +-- 主从复制 <-- Master
```

## 二、实现方式

### 2.1 MySQL 主从复制

* **master.cnf**：server-id=1, log-bin 开启
* **slave.cnf**：server-id=2, read_only=1
* **slave-init.sh**：从库首次启动时，连接主库获取 binlog 位置，配置复制

### 2.2 代码侧

* **dynamic-datasource**：master 数据源连接主库，slave 数据源连接从库
* **@DS("master")**：注册等写操作
* **@DS("slave")**：登录、商品查询等读操作

## 三、首次部署

**注意**：读写分离使用新的 volume（mysql_master_data、mysql_slave_data），若此前有旧数据，需清理：

```powershell
docker compose down -v
docker compose up -d --build
```

## 四、验证

```powershell
# 1. 注册（写 master）
curl.exe -X POST http://localhost/api/users/register -H "Content-Type: application/json" -d "{\"username\":\"test123\",\"password\":\"123456\",\"phone\":\"13800138000\"}"

# 2. 登录（读 slave）
curl.exe -X POST http://localhost/api/users/login -H "Content-Type: application/json" -d "{\"username\":\"test123\",\"password\":\"123456\"}"

# 3. 查看从库复制状态
docker exec seckill-mysql-slave mysql -u root -p050612 -e "SHOW SLAVE STATUS\G"
```

若 Slave_IO_Running 和 Slave_SQL_Running 均为 Yes，则主从复制正常。
